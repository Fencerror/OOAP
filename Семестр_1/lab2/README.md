
# Диаграмма Классов
## Текстовое описание
1. Абстрактный класс "Staff"  
Поля:
- id: int — уникальный идентификатор сотрудника, используемый для различения между сотрудниками.
- name: String — полное имя сотрудника.
- position: String — должность сотрудника, определяющая его обязанности в библиотеке.

Операции:
- work(): void — выполняет повседневные задачи и обязанности, соответствующие должности сотрудника.



2. Класс "LibraryStaff" (наследуется от "Staff")  
Поля:
- supervisor: LibraryStaff — ссылка на руководителя, если у сотрудника есть прямой начальник в библиотеке.

Операции:
- serveReader(): void — помогает читателям в библиотеке, предоставляет информацию и обслуживание.
- manageReadingRoom(): void — отвечает за организацию и поддержание порядка в читальном зале, включая управление резервированием мест.
- getCopies(): string[*] — получение списка копий книг.
- checkAvailability: boolean — логика обработки в случае доступности книги.
- identificationAnalysis: boolean  — анализ идентификации читателя.
- getReaderPreferences: string[*] — получение предпочтений читателя.
- readerCheckUp: boolean — проверка доступности книги на основе предпочтений читателя и идентификатора, выдача книги.
- BookRezervation():int
- chooseBookCopy():int
- getRoom():int
- updteRezervtionRequest():int


3. Интерфейс "IReservable"  
Операции:
- reserve(reader: Reader): void — зарезервировать элемент для определенного читателя.
- cancelReservation(reader: Reader): void — отменить резервирование элемента для указанного читателя.
- isReserved(): boolean — возвращает информацию о том, зарезервирован ли элемент в данный момент.



4. Класс "Book" (реализует интерфейс "IReservable")  
Поля:
- id: int — уникальный идентификатор книги в библиотеке.
- title: String — название книги.
- author: String — имя автора книги.
- reservedBy: Reader — читатель, зарезервировавший книгу, если резервирование активно.

Операции:
- reserve(reader: Reader): void — резервирует книгу для указанного читателя, если она доступна.
- cancelReservation(reader: Reader): void — отменяет резервирование книги, освобождая ее для других читателей.
- isReserved(): boolean — проверяет, зарезервирована ли книга на текущий момент.



5. Класс "Category"  
Поля:
- name: String — название категории, описывающее тип или жанр книг (например, "Научная фантастика", "История").



6. Класс "Reader"  
Поля:
- id: int — уникальный идентификатор читателя, используемый для различения пользователей.
- name: String — полное имя читателя.

Операции:
- placeOrder(book: Book): Order — создает заказ для резервирования книги и возвращает объект заказа.
- cancelOrder(order: Order): void — отменяет указанный заказ, если читатель передумал.


7. Класс "Order"  
Поля:
- orderNumber: int — уникальный номер заказа, используемый для его отслеживания.

Операции:
- confirm(): void — подтверждает заказ, окончательно закрепляя книгу за читателем.
- cancel(): void — отменяет заказ, освобождая книгу для других читателей.



8. Класс "Library"  
Поля:
- id: int — уникальный идентификатор библиотеки.
- name: String — название библиотеки, например, "Центральная библиотека".



9. Класс "Room"  
Поля:
- id: int — уникальный номер комнаты в библиотеке.
- name: String — название комнаты, например, "Зал для чтения" или "Конференц-зал".

Операции:
- open(): void — открывает комнату для посещения.
- close(): void — закрывает комнату, делая ее недоступной для использования.



10. Класс "ReadingRoom" (наследуется от "Room")  
Поля:
- capacity: int — максимальное количество людей, которые могут находиться в читальном зале одновременно.

Операции:
- reserveSeat(reader: Reader): void — резервирует одно из доступных мест для указанного читателя.
- releaseSeat(reader: Reader): void — освобождает место, ранее зарезервированное для читателя.
- manageRoom(): void — выполняет функции управления читальным залом, такие как контроль наличия мест и обслуживание читателей.


11. Класс "Seat" (реализует интерфейс "IReservable")  
Поля:
- seatNumber: int — уникальный номер места в читальном зале.
- isAvailable: boolean — флаг, показывающий, доступно ли место для резервирования.
- reservedBy: Reader — читатель, зарезервировавший место, если резервирование активно.

Операции:
- reserve(reader: Reader): void — резервирует место для указанного читателя, если оно доступно.
- cancelReservation(reader: Reader): void — отменяет резервирование места, освобождая его.
- isReserved(): boolean — проверяет, зарезервировано ли место в данный момент.

# Диаграмма(мы) последовательностей

## SD1
### 1. Диаграмма последовательности для резервирования книги  
   Описывает процесс, когда читатель резервирует книгу в системе. Основные шаги:
   1. Читатель вызывает метод на резервирование книги.
   2. Сотрудник проверяет доступность книги.
   3. Если книга доступна, проверяется номер зала.
   4. Создается объект Reservation, и состояние книги обновляется на "зарезервирована".
   5. Обновляется состояние заявки на резервирование.

## SD2
### 2. Диаграмма описывает процесс проверки доступности экземпляра книги. Основные шаги включают:
   1. Сотрудник бииблиотеки запрашивает доступность книги.
   2. Система возвращает список подходящих экземпляров.
   3. Для каждого экземпляра проверяется, доступен ли он для выдачи читателю.
   4. Если да, то возвращается true, если нет false.

## SD3
### 3. Эта диаграмма описывает процесс обслуживания читателя. Основные шаги включают:
1. Читатель приходит к библиотекарю.
2. Библиотекарь идентифицирует читателя (талон и проч.)
3. Проводится анализ читателя. (должен вернуть, утери, проч.)
4. Библиотекарь запрашивает какая книга нужна.
5. Если требуемая книга требует обновления, то она обновляется.
6. Проводится поиск нужной книги.
7. Если экземпляр книги не тот, обновляются предпочтения и экземпляр книги.
